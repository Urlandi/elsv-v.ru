Планы

  1. Карта - расширение функционала, упрощение и автоматизация
  2. Интерактивное поведение - возможность перехода к страницам информации с провайдерами, всплывающие комментарии, масштабирование
    1. Правка непосредственно из браузера с последующей генерацией формализированного представления в DOT
    2. Отображение оперативной информации по провайдерам, например, загруженность каналов в данный момент (конечно такое возможно только при непосредственном сотрудничестве)
  3. Автоматическая генерация aka subnets.ru или radar.qrator.net для компаний на основе информации помещенной в Каталог, базы данных RIPE и активной маршрутной информации.
  4. Каталог - больше технической информации и её анализа.
  5. Форма для первоначального ввода информации и её изменения и отслеживания изменений. С поддержкой синтаксиса Wiki и возможностью последующей правки каталога в Wiki стиле
  6. Определение суммарных карт и отчётов в удобном представлении для зон покрытия наложение на карту районов города (OSM?), проводного и беспроводного подключения.
    1. Разработка механизмов получения достоверной информации о зонах покрытия с указанием зоны охвата Wi-Fi, возможно как wifi4free, но с большим количеством информации по охвату, не только адрес, и привязкой к провайдеру.
    2. Автоматические отчёты - общее количество провайдеров, оценочное количество подключений (например по количеству используемых IP адресов и технологии подключения), оценочные показатели качества, возможно рейтинг.
  7. Получение официальных данных от заинтересованных компаний провайдеров - используемое оборудование, мощности, технологии
  8. whois интерфейс к собранной базе провайдеров, геолокация по адресам
  9. Инструменты - максимальная интеграция для получения самой актуальной и достоверной информации, для широкого применения в решении задач по предыдущим пунктам
  10. Разработка инструментов (агентов) для использования со стороны пользователя услуг для получения оперативной информации о подключении, связности, качестве. Стремление разработки безустановочных средств, в идеале, опираясь только на возможности самого браузера.
    1. Разработка средств для мобильных устройств - Android, iOS
    2. Разработка средств для встраивания в маршрутизаторы и другие сетевые устройства
    3. Разработка собственного устройства агента
  11. API для возможности интеграции сторонних приложений
  12. Online средства - возможность всё проверить
  13. Стандартные сетевые средства для запуска непосредственно со страниц сайта - traceroute, ping, dig
    1. Объединённый lg, для всех провайдеров предоставляющий данный сервис с единой точкой входа, на основе LG
  14. Удалённый доступ к Windows или *nix shell, с большими возможностями по удалённой диагностике
  15. Мониторинг работы сайта - количество запросов, статистика трафика, количество сессий
  16. Сайт - гибкость
  17. Актуальная информация за счёт автоматического поиска и подачи информации - использование новостного робота агрегатора
    • Частично реализован агрегатор новостей с использованием RSSReader. Отображается только текущее состояние ленты, истории - нет, поиска - нет, время выводится в исходном формате. События относятся теперь к действительно к значимым событиям, которые необходимо запоминать и хранить, или новости непосредственно генерируемые сайтом.
    • Полностью реализовано. Лента отображает последние 10 событий. Для каждого дня используется MonthCalendar, отображаются все события за день.
  18. Новости провайдеров из Каталога и отрасли в текущем регионе, RSS лента, возможность получить сведения обо всех текущих работах и авариях в одном месте.
  19. Расширение за счёт плагинов, (миграция части функций на Django)
  20. Привлечение заинтересованных лиц к работе в проекте
  21. Добавление социальных средств общения и разработки, возможно с использованием Diaspora* и Apache Bloodhound
  22. Привлечение представителей компаний провайдеров, редакторов, для публикации пресс релизов, статей, планируемых изменениях.
